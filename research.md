# 영상 대화 내용 JSON 대사집 변환 프로그램 연구 보고서

본 보고서는 영상 파일에서 대화 내용을 추출하여 화자가 구분된 JSON 형식의 대사집으로 변환하고, 이를 설치 과정 없이 다양한 환경에서 실행하기 위한 전략을 정리합니다.

## 1. 핵심 구현 전략 (Pure STT & Script)

### 1.1 기술 스택 (기존 `stt.md` 기반 최적화)
*   **음성 인식 (STT):** `faster-whisper`
    *   **이유:** OpenAI의 Whisper 모델을 C++로 최적화하여 매우 빠르며, 낮은 사양의 컴퓨터에서도 작동 가능합니다. (대사집의 텍스트 정확도 보장)
*   **화자 분리 (Diarization):** `pyannote.audio`
    *   **이유:** 누가 언제 말했는지 구분하여 "대사집"의 형태를 갖추게 합니다.
*   **미디어 처리:** `FFmpeg`
    *   **이유:** 다양한 영상 포맷(mp4, mkv, mov 등)에서 오디오를 안정적으로 추출합니다.

### 1.2 배포 전략 및 연산 장치(GPU/CPU) 최적화
사용자가 어떤 환경에서도 프로그램을 실행할 수 있도록 배포와 하드웨어 설정을 구성합니다.

1.  **연산 장치 선택 (Device Selection):**
    *   **GPU 모드:** NVIDIA 그래픽카드가 있는 경우 사용 (CUDA 가속). 대사집 생성 속도가 비약적으로 빠름.
    *   **CPU 모드:** 그래픽카드가 없거나 사양이 낮은 환경용. 속도는 느리지만 호환성이 완벽함.
    *   **GUI 적용:** 사용자가 프로그램 시작 전 버튼 하나로 CPU/GPU를 선택할 수 있게 구현.
2.  **CUDA 설정 가이드:**
    *   **자동 설정:** 프로그램 실행 시 `ctranslate2`(Whisper 엔진)에 필요한 필수 DLL들을 패키지 내부에 포함하여 사용자가 별도로 CUDA를 설치하지 않아도 작동하게 구성 가능.
    *   **수동 설정 시:** NVIDIA 드라이버 설치 -> CUDA Toolkit 설치 -> cuDNN 라이브러리를 CUDA 폴더에 복사하는 과정이 필요함.
3.  **단일 실행 파일 (.exe) 제작:**
    *   `PyInstaller` 등을 사용하여 모든 의존성을 하나의 실행 파일로 빌드. 모델 파일(base, small 등)을 내장하여 인터넷 연결 없이도 즉시 실행 가능하게 만듦.

## 2. GUI (사용자 인터페이스) 및 사용자 경험

사용자가 명령어를 입력하지 않고 마우스 클릭만으로 작업할 수 있도록 GUI를 추가합니다.

### 2.1 추천 GUI 프레임워크
*   **CustomTkinter:** 가장 가볍고 윈도우 스타일의 세련된 UI를 만들 수 있어 배포 용량을 줄이는 데 유리합니다.
*   **Flet:** 모던한 디자인과 드래그 앤 드롭 기능을 매우 쉽게 구현할 수 있습니다.

### 2.2 GUI 주요 기능
*   **장치 선택 스위치:** GPU(NVIDIA)와 CPU 중 하나를 선택하는 기능.
*   **드래그 앤 드롭 (Drag & Drop):** 영상 파일을 창으로 끌어다 놓으면 즉시 인식.
*   **실시간 진행률 표시:** "오디오 추출 중...", "화자 분리 중...", "대사집 생성 중..." 등의 상태를 시각적으로 표시.
*   **결과 미리보기 창:** 변환된 내용을 JSON 저장 전에 화면에서 미리 확인하고, 화자 이름(예: 화자_01 -> 홍길동)을 수정할 수 있는 편집 기능.

## 3. 대사집 특화 JSON 구조 제안

단순 텍스트 나열이 아닌, 편집과 활용이 용이한 구조입니다.

```json
{
  "video_info": {
    "filename": "interview_01.mp4",
    "duration": "00:05:30"
  },
  "script": [
    {
      "index": 1,
      "speaker": "화자_A",
      "start_time": "00:00:01.500",
      "end_time": "00:00:04.200",
      "text": "안녕하세요, 오늘 인터뷰에 응해주셔서 감사합니다."
    },
    {
      "index": 2,
      "speaker": "화자_B",
      "start_time": "00:00:04.500",
      "end_time": "00:00:06.800",
      "text": "별말씀을요. 초대해주셔서 제가 더 기쁩니다."
    }
  ]
}
```

## 4. 대사집 제작에 유용한 추가 기능 제안

LLM 없이도 대사집의 완성도를 높여주는 실용적인 기능들입니다.

### 3.1 정확도 및 가독성 향상
*   **오디오 노이즈 제거 (Denoiser):** 배경 음악이나 잡음이 섞인 영상에서 목소리만 정밀하게 추출하여 인식률을 높임.
*   **문장 단위 자동 끊기:** 너무 긴 문장을 의미 단위나 일정 시간 단위로 나누어 가독성 좋은 대사집 구성.
*   **단어별 타임스탬프:** JSON 내에 단어 하나하나가 나오는 정확한 시간을 포함하여, 특정 단어를 클릭하면 영상의 해당 위치로 이동하는 기능의 기반 마련.

### 3.2 사용자 편의 기능
*   **다양한 포맷 출력:** JSON뿐만 아니라 자막 파일(`.srt`, `.vtt`), 텍스트 파일(`.txt`), 엑셀(`.xlsx`) 등으로 동시 저장.
*   **화자 이름 일괄 변경:** "화자_01"로 추출된 이름을 사용자가 "홍길동"으로 입력하면 전체 대사집에서 일괄 변경하는 기능.
*   **진행률 표시기 (Progress Bar):** 변환 작업이 얼마나 남았는지 콘솔이나 GUI를 통해 실시간으로 확인.

### 3.3 관리 및 검색
*   **내장 검색 도구:** 대사집 내 특정 키워드를 검색하면 해당 시간이 포함된 구간을 즉시 찾아주는 기능.
*   **자동 인코딩 감지:** 영상의 언어를 자동으로 감지하여 적절한 STT 모델 매개변수 설정.

## 4. 향후 개발 로드맵

1.  **Phase 1:** `faster-whisper`와 `pyannote`를 연동하여 화자 구분이 포함된 JSON 추출 CLI 구현.
2.  **Phase 2:** 윈도우 환경에서 DLL 오류 없이 실행되도록 Portable 환경 구성 및 테스트.
3.  **Phase 3:** 사용자 편의를 위해 자막 파일(SRT) 동시 생성 및 화자 이름 매핑 기능 추가.
4.  **Phase 4:** (선택 사항) 간단한 GUI를 추가하여 드래그 앤 드롭으로 작업을 시작할 수 있게 개선.
